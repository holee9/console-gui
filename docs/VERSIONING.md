# HnVue 문서 버전 관리 정책

이 문서는 HnVue 프로젝트의 규제 문서 버전 관리 정책을 정의합니다.
IEC 62304:2006+AMD1:2015 및 FDA 21 CFR 820.30 요구사항을 준수합니다.

## 버전 번호 체계

**형식**: `vX.Y` (주버전.부버전)

| 변경 유형 | 버전 변경 | 예시 | 승인 필요 |
|-----------|----------|------|----------|
| 중대 변경 (인허가 재심사 필요) | 주버전 증가 | v1.0 → v2.0 | RA/QA 책임자 + PM |
| 오류 수정, 명확화 | 부버전 증가 | v1.0 → v1.1 | 작성자 + 검토자 |
| 초안 작성 중 | Draft 표시 | v1.0 (Draft) | 해당 없음 |

## 파일 명명 규칙

```
DOC-{번호}_{제목}_v{주버전}.{부버전}.md
```

예시:
- `DOC-005_SRS_v2.0.md` — 소프트웨어 요구사항 명세서 v2.0
- `DOC-032_RTM_v2.0.md` — 요구사항 추적 매트릭스 v2.0

## 주버전 증가 트리거

다음 변경 시 주버전을 증가시킵니다:
- 요구사항 추가/삭제/중대 변경 (SRS, FRS)
- 아키텍처 구조 변경 (SAD, SDS)
- 위험 분류 변경 (FMEA, RMP)
- 새로운 SOUP 컴포넌트 추가 (DOC-033)
- 인허가 제출 이후의 모든 변경

## 부버전 증가 트리거

다음 변경 시 부버전을 증가시킵니다:
- 오탈자 수정
- 표현 명확화 (의미 변경 없음)
- 참조 문서 링크 업데이트
- 테스트 결과 업데이트 (합격 상태 유지)

## 아카이브 정책

- 이전 버전은 `docs/archive/` 디렉토리에 보관합니다
- 아카이브 파일명: `DOC-{번호}_{제목}_v{이전버전}.md`
- 최소 2개 이전 버전을 유지합니다 (규제 감사 대비)
