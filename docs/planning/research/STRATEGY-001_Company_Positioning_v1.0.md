## 1. 회사 현황 냉정 분석 (Honest Assessment)

### 1.1 강점 (Strengths)

- **CsI 증착 기술 내재화** (scintillator 자체 생산): 전 세계적으로 CsI scintillator를 자체 증착할 수 있는 업체는 극소수 (Rayence, Hamamatsu, Trixell, Carestream 등)
- **FPD 설계/제조 역량** (하드웨어 수직 통합): detector 설계부터 생산까지 자체 수행
- **CsI → FPD → Console 전체 밸류체인 가능성**: Rayence와 유사한 수직통합 구조를 추구할 수 있는 기반 보유
- **소규모 조직의 빠른 의사결정**: 100명 미만 조직으로 시장 변화에 민첩한 대응 가능
- **HW/기구 개발 역량**: HW 3명 + 기구 1명으로 detector 하드웨어 자체 개발 가능

### 1.2 약점 (Weaknesses) -- 냉정하게

**SW 개발 역량 심각한 부족:**
- SW 개발 인력 **단 2명** (업계 최소 수준, 사실상 최하위)
  - 비교: DRTECH ~20명 SW팀 추정 (EConsole1 + TRUVIEW ART + DEPAI AI 등 다수 제품 운영)
  - Rayence ~15명 (Xmaru View V1 + VetView + PodView 등 다수 변형 제품, 18개국어 지원)
  - Vieworks ~10명 (VXvue + PureImpact 알고리즘, FDA 510(k) 인증 완료)
  - JPI ~5명 (ExamVue + Cloud PACS)
  - Siemens/GE/Philips: 수백~수천명 규모
- **의료기기 SW 개발 경험 부재** (추정): IEC 62304 기반 SW 개발 라이프사이클 경험이 팀 내에 없을 가능성 높음
- **IEC 62304 / FDA / CE 규제 대응 역량 미확인**: Class B 이상 의료기기 SW 인증을 받아본 내부 인력 부재
- **UI/UX 전문 인력 부재**: 의료 영상 소프트웨어의 사용성은 임상 환경에서 생사를 가르는 요소. 전담 UX 디자이너 없음
- **QA/테스트 전담 인력 부재**: IEC 62304 Class B는 체계적 V&V(Verification & Validation) 필수. 개발자가 자체 테스트하는 구조로는 규제 통과 불가
- **DICOM 전문 역량 부재** (추정): DICOM은 3,000페이지 이상의 표준으로, 숙련된 엔지니어도 완전한 구현에 수개월 소요
- **경쟁사 대비 브랜드 인지도 부족**: DRTECH(8,000+ 설치기반), Rayence(전 제품 라인업), Vieworks(수상 실적) 대비 시장 인지도 열위
- **글로벌 설치 기반 미미**: 경쟁사들은 수천~수만 대 설치 실적 보유

### 1.3 기회 (Opportunities)

- **FPD 번들 SW로 디텍터 경쟁력 강화**: 업계 표준이 "detector + console SW 번들 무료 제공"으로 수렴 중. Console SW 없이는 FPD 판매 자체가 어려움
- **CsI+FPD 수직통합 업체가 소수**: Rayence, Hamamatsu, Trixell 정도만 scintillator부터 detector까지 수직통합. 이 희소 가치를 SW로 완성해야 함
- **중소 병원/개발도상국 시장 성장**: 아시아-태평양 지역이 디지털 방사선 시장에서 가장 빠르게 성장 중 (~20% 점유, 최고 CAGR)
- **AI/클라우드 트렌드로 SW 가치 상승**: AI X-ray 영상 SW 시장 CAGR 15.58% (IMARC Group 기준, 2033년 $949.6M 전망)
- **OR Technology OEM 화이트라벨 활용 가능**: dicomPACS DX-R을 브릿지 솔루션으로 도입하여 즉시 시장 진입 후 자체 개발 병행 가능
- **오픈소스 기반 가속**: iBEX (학술 검증된 DR 오픈소스), fo-dicom 5.x (MIT 라이선스 DICOM 라이브러리) 등 활용 가능
- **디지털 방사선 SW 세그먼트 성장**: SW 컴포넌트가 DR 시장 내 가장 높은 CAGR (~6.6%) 기록 중

### 1.4 위협 (Threats)

- **경쟁사 모두 자체 Console SW 보유** (table stakes): Rayence(Xmaru View), Vieworks(VXvue), DRTECH(EConsole1), JPI(ExamVue) 등 모든 주요 경쟁사가 성숙한 자체 SW 보유. Console SW는 "있으면 좋은 것"이 아니라 **"없으면 팔 수 없는 것"**
- **Samsung + Lunit AI 통합 가속**: 대기업의 AI 통합으로 중소업체의 기술 격차 확대
- **DRTECH DEPAI/TRUVIEW ART**: 경쟁사가 AI 기반 영상처리(DEPAI)와 MTF 30% 향상 기술(TRUVIEW ART) 등 고급 기능으로 차별화 가속
- **FDA 사이버보안 규제 강화**: 2023년 PATCH Act 이후 premarket submission에 사이버보안 요건 의무화. 소규모 업체에 불균형적 부담
- **대형 OEM의 중저가 시장 진출**: Siemens, GE, Philips의 이머징 마켓 진출로 가격 경쟁 심화
- **인력 이탈 리스크**: SW 2명 중 1명이라도 퇴사하면 프로젝트 중단 위기
- **멀티벤더 대응 압력**: OR Technology, Examion(7,000+ 설치기반) 등이 멀티벤더 지원으로 시장 장악. 단일 벤더 전용 SW는 시장 확장성 제한

### 2.2 핵심 현실 인식

**냉정한 사실:**
1. 경쟁사들은 최소 5-10명의 SW 전담팀 + **수년~십수년간의 개발 이력** 보유
2. DRTECH은 20년 이상의 이미징 기술 전문성, Rayence는 세계 최초 dental/medical/vet/industrial 풀라인업, Vieworks는 수상 실적의 VXvue 보유
3. **2명의 SW 인력으로 경쟁사 수준의 풀 Console을 자체 개발하는 것은 비현실적**
4. 그러나 Console SW가 없으면 FPD 판매 시 **경쟁 열위 확정** (번들 기대가 업계 표준)
5. 모든 주요 한국 FPD 업체가 자체 Console SW를 보유한 상황에서, SW 없는 FPD는 "반제품" 취급

**결론:** 풀스펙 자체 개발은 불가능하지만, 최소한의 경쟁력 있는 Console SW는 **생존 필수 요건**

### 2.3 독자적 포지셔닝 기회

우리만의 차별화 가능 영역:
- **CsI 특성 최적화 영상 처리**: 자사 CsI scintillator의 물리적 특성을 완벽히 이해하고 있으므로, detector-specific 최적화가 가능. 경쟁사 대비 **유일하게 CsI 증착부터 영상 처리까지 최적화** 가능
- **FPD 하드웨어 심층 통합**: 자사 detector와의 low-level 통신 최적화 (calibration, 상태 모니터링, 온도 보정 등)
- **소규모 시장/니치 타겟팅**: 대형 경쟁사가 관심 없는 개발도상국, 소규모 클리닉, 레트로핏 시장

## 3. Console SW 개발 전략 옵션 분석

### 3.1 전략 옵션 비교

| 옵션 | 설명 | 소요 기간 | 비용 | 위험 | 차별화 |
|------|------|----------|------|------|--------|
| **A. 풀 자체 개발** | MRD 69개 요구사항 전체 구현 | 3-4년 | 높음 (추가 채용 필수, $2M-$30M 업계 기준) | 매우 높음 | 높음 |
| **B. OEM 화이트라벨** | OR Tech dicomPACS DX-R 도입 | 3-6개월 | 라이선스 비용 (연간) | 낮음 | 낮음 (차별화 어려움) |
| **C. 오픈소스 기반** | iBEX + fo-dicom 기반 자체 개발 | 18-24개월 | 중간 | 높음 (규제 리스크, iBEX는 R&D용) | 중간 |
| **D. 하이브리드** (권장) | 핵심 획득 모듈 자체 개발 + 범용 기능 오픈소스/외주 | 12-16개월 | 중간 | 중간 | 중-높음 |

### 3.2 각 옵션 상세 분석

#### Option A: 풀 자체 개발 -- 비현실적

- 업계 기준 Class II 의료기기 SW 풀 개발: $2M-$30M, 1-3년 (일반적 팀 규모 기준)
- HIPAA compliance만 추가 15-25% 비용 증가 ($15K-$75K)
- **2명으로는 3-4년이 아니라 5-7년 이상 소요될 가능성**
- DRTECH이 20년간 축적한 기술력을 2명이 따라잡는 것은 불가능

#### Option B: OEM 화이트라벨 -- 브릿지 솔루션으로 고려

- OR Technology dicomPACS DX-R: 멀티벤더 지원, 터치스크린 UI, GLI(가상 산란격자), AI 리포팅 옵션
- 7,000+ 시스템에 검증된 솔루션 (Examion 파트너 네트워크)
- **장점**: 즉시 시장 진입, 규제 부담 최소화 (OR Tech 측 인증 활용 가능성)
- **단점**: 라이선스 비용 지속, 자사 FPD 최적화 제한, 차별화 불가, 벤더 종속

#### Option C: 오픈소스 기반 -- 규제 리스크

- iBEX: 학술 논문으로 검증(J Digital Imaging, 2020), 모듈형 구조, 플러그인 확장
- **치명적 한계**: "Freeware for R&D" 명시, 상용 의료기기 배포 시 별도 규제 작업 필수
- fo-dicom 5.x: MIT 라이선스로 상용 사용 자유, 순수 .NET, DICOM 네트워킹 내장
- **결론**: fo-dicom은 활용 가능, iBEX는 참조 수준

#### Option D: 하이브리드 접근 -- 권장

**원칙: "플랫폼은 사고, 차별화는 만든다" (Buy the Platform, Build the Differentiator)**

자체 개발 영역 (2명 SW 집중):
- **FPD 하드웨어 인터페이스** (자사 디텍터 최적화 -- 이것은 우리만 할 수 있음)
- **CsI 특성 맞춤 영상 처리 파이프라인** (핵심 차별화 요소)
- **촬영 워크플로우 UI/UX** (임상 사용자 접점)
- **Generator 통신 프로토콜** (하드웨어 통합)

외부 조달/오픈소스 영역:
- DICOM 통신: fo-dicom 5.x (MIT 라이선스, 검증된 .NET DICOM 라이브러리)
- 데이터베이스: SQLite + EF Core (제로 설정, 내장형)
- 테스트 PACS: Orthanc (GPLv3, 개발/테스트 전용 별도 서비스로 사용)
- IEC 62304 문서: 외주 (규제 컨설팅 전문 업체)
- UI 프레임워크: WPF (.NET 8)

### 3.3 브릿지 전략 (Option B + D 병행)

**단기 (0-6개월)**: OR Technology dicomPACS DX-R OEM 도입으로 즉시 FPD 번들 판매 가능
**중기 (6-16개월)**: 자체 HnVue Console SW 개발 (Option D 하이브리드)
**장기 (16개월+)**: OEM에서 자체 SW로 전환, OEM 라이선스 종료

이 접근의 장점:
- FPD 판매를 SW 개발 완료까지 기다리지 않아도 됨
- OEM 사용 중 임상 현장 피드백 수집 가능 (자체 SW 개발에 반영)
- 매출 확보와 개발 병행

## 4. 현실적 요구사항 범위 재정의

### 4.1 MRD 69개 요구사항 현실 필터링

현 MRD의 69개 요구사항을 회사 현실에 맞게 재분류:

| 구분 | 정의 | 해당 MR 수 | Phase |
|------|------|:----------:|:-----:|
| **Core** | FPD 판매에 반드시 필요한 최소 기능 | ~20개 | Phase 1 (Y1) |
| **Regulatory** | 인허가에 필수인 규제 요건 | ~10개 | Phase 1 (Y1) |
| **Competitive** | 경쟁사 대비 필요하나 후순위 가능 | ~15개 | Phase 2 (Y2) |
| **Aspirational** | 이상적이나 현 규모에서 비현실적 | ~24개 | Phase 3+ 또는 제외 |

### 4.2 Phase 1 Core Requirements (SW 2명, 12-16개월)

#### 워크플로우 (필수)
- **MR-001** DICOM MWL 환자 조회 (기본)
- **MR-002** 촬영-PACS 전송 30초 이내
- **MR-003** 5클릭 워크플로우
- **MR-004** 기본 촬영 프로토콜 (**50개로 축소**, 원래 200 -> 50)
- **MR-005** 즐겨찾기/사용자 설정 (기본 UX)
- **MR-007** 응급 검사 처리 (임상 필수)
- **MR-009** MPPS 자동 보고
- **MR-010** 디텍터 상태 표시 (자사 FPD 전용)

#### 영상 품질 (필수)
- **MR-011** 자동 영상 처리 (CsI 최적화 -- **핵심 차별화**)
- **MR-012** 기본 영상 조작 (W/L, Zoom, Pan, Annotation)
- **MR-013** 엣지 강조/노이즈 감소 프리셋

#### 통합 (필수)
- **MR-019** DICOM 핵심 서비스 (C-STORE, MWL, MPPS)
- **MR-020** IHE SWF 프로필
- **MR-021** 3개 이상 PACS 호환 테스트
- **MR-025** 워크리스트 자동 수신

#### 선량 관리 (필수)
- **MR-027** RDSR 생성/전송
- **MR-031** AEC 모니터링

#### 사이버보안 (규제 필수)
- **MR-033** RBAC 기본 (관리자/사용자 2단계)
- **MR-034** PHI 암호화 (AES-256, .NET 내장 API 활용)
- **MR-035** 감사 로그 (Serilog 기반)
- **MR-036** SBOM 생성 (도구 활용)
- **MR-037** CVD 프로세스 (절차 수립)
- **MR-039** 코드 서명
