## 1. 문서 정보

### 1.1 문서 목적

IEC 62304(SW 수명주기)와 ISO 14971(위험 관리)의 사이버보안 확장 표준인 IEC 81001-5-1:2021을 기준으로 HnVue Console SW (HnVue)의 사이버보안 활동 전체가 표준을 준수함을 체계적으로 문서화한다. 본 문서는 개별 기술 문서(그룹 C)를 표준 조항에 매핑하는 **준수 매트릭스(Compliance Matrix)** 역할을 한다.

### 2.2 Full Compliance vs Annex F (Transitional) 선택

IEC 81001-5-1은 신제품에 대한 전체 준수(Full Compliance)와, 기인허가 제품에 대한 전환 조항(Annex F: Transitional Provisions)을 구분한다.

| 제품 | 선택 경로 | 근거 |
|------|----------|------|
| **HnVue Console SW (신규)** | **Full Compliance** | 신규 개발 제품으로 처음부터 IEC 81001-5-1 요건을 반영하여 설계 |
| 기인허가 Detector SW (해당 시) | Annex F (전환 경로) 검토 | 기존 허가 제품은 Annex F §F.2–F.5의 전환 의무 사항만 이행 후 단계적 완전 준수 이행 계획 수립 |

**HnVue Full Compliance 선택 근거**:
1. 신규 개발 제품으로 기존 허가 제약 없음
2. EU MDR Notified Body는 Full Compliance를 권장하며, Annex F는 심사 과정에서 추가 질의 대상이 됨
3. DOC-016(CMP)을 포함한 사이버보안 문서 체계가 Full Compliance 요건을 대부분 충족하도록 설계됨
4. FDA §524B 요건과 IEC 81001-5-1 Full Compliance 요건이 상당 부분 중첩되므로 통합 대응 가능

## 3. 보안 역량 수준 (Security Capability Level, SCL)

### 3.1 SCL 선택

| 항목 | 내용 |
|------|------|
| 선택 SCL | **SCL 2** |
| 선택 근거 | 병원 내부 네트워크(Intranet) 환경에서 운용되며, 직접적 인터넷 노출은 없으나 PACS/RIS/HIS 등 복수 시스템과 연동됨. DOC-016 §2 범위에 명시된 위협 행위자 프로파일은 기회주의적 공격자(Opportunistic Attacker) 및 내부 위협(Insider Threat)으로, SCL 2 수준의 방어가 적절함. SCL 3은 국가 수준(Nation-State) 위협 행위자 대상으로 HnVue 사용 환경 대비 과도함. |

> **SCL 정의**:
> - SCL 1: 기본 보안 — 의도치 않은 또는 우연한 사이버보안 사건에 대한 보호
> - SCL 2: 중간 보안 — 단순한 수단과 저수준 동기를 가진 의도적 위협 행위자에 대한 보호
> - SCL 3: 고급 보안 — 정교한 수단과 보통 수준의 자원 및 동기를 가진 위협 행위자에 대한 보호

### 3.2 SCL 2 선택 근거 상세

| 위협 환경 요인 | 분석 내용 | SCL 2 적합성 |
|--------------|----------|-------------|
| 네트워크 노출 | 병원 내부 인트라넷 전용, 인터넷 직접 연결 없음 | 적합 |
| 연결 시스템 | PACS, RIS, HIS, Generator 제어 (4개 인터페이스) | 적합 |
| 데이터 민감도 | 환자 PHI(Protected Health Information) 포함 DICOM 데이터 | 적합 (SCL 3 불필요) |
| 규제 환경 | FDA §524B, EU MDR Annex I §21 — SCL 3 명시적 요구 없음 | 적합 |
| 운용 환경 | 병원 IT 관리 부서가 관리하는 통제된 환경 | 적합 |
