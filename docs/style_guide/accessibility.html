Accessibility Guidelines

WCAG 2.2 compliance and medical device accessibility

Overview

HnVue UI is designed to meet WCAG 2.2 Level AA requirements and medical device accessibility standards. This ensures our application is usable by all healthcare professionals, including those with disabilities.

Regulatory Requirements

WCAG 2.2 Level AA

Web Content Accessibility Guidelines for web-based interfaces

Section 508

US federal accessibility requirements for medical devices

EN 301 549

European accessibility requirements for ICT products

FDA Human Factors

Accessibility considerations in medical device usability

Keyboard Navigation

Tab Order

All interactive elements must be keyboard accessible with a logical tab order.

Do

Follow visual left-to-right, top-to-bottom order for tab navigation

Don't

Use random tab orders that confuse keyboard users

Keyboard Shortcuts

Provide keyboard alternatives for all mouse actions.

Action Keyboard Shortcut
Next control Tab
Previous control Shift + Tab
Activate button Space or Enter
Close modal/dialog Esc
Navigate to Worklist Alt + 1
Navigate to Studylist Alt + 2
Navigate to Acquisition Alt + 3
Open search Ctrl + F
Save Ctrl + S
Help F1

XAML Keyboard Navigation

XAML 
<!-- Set Tab Order -->
<StackPanel>
    <!-- TabIndex 0-99 for flow order -->
    <TextBox TabIndex="0"
             Text="{Binding PatientId}"/>
    <TextBox TabIndex="1"
             Text="{Binding PatientName}"/>
    <TextBox TabIndex="2"
             Text="{Binding DateOfBirth}"/>

    <!-- Negative TabIndex removes from tab order -->
    <TextBlock Text="Read-only label"
               TabIndex="-1"/>
</StackPanel>

<!-- Keyboard Access Key -->
<Button Content="_Save"
        Command="{Binding SaveCommand}">
    <Button.InputBindings>
        <KeyBinding Key="S"
                    Modifiers="Control"
                    Command="{Binding SaveCommand}"/>
    </Button.InputBindings>
</Button>

<!-- Focus Visual Indicator -->
<Style TargetType="TextBox">
    <Setter Property="Template">
        <Setter.Value>
            <ControlTemplate TargetType="TextBox">
                <Border Name="Border"
                        Background="{TemplateBinding Background}"
                        BorderBrush="{TemplateBinding BorderBrush}"
                        BorderThickness="{TemplateBinding BorderThickness}">
                    <ScrollViewer x:Name="PART_ContentHost"/>
                </Border>
                <ControlTemplate.Triggers>
                    <Trigger Property="IsKeyboardFocusWithin" Value="True">
                        <Setter TargetName="Border"
                                Property="BorderBrush"
                                Value="{StaticResource State.Focus}"/>
                        <Setter TargetName="Border"
                                Property="BorderThickness"
                                Value="2"/>
                    </Trigger>
                </ControlTemplate.Triggers>
            </ControlTemplate>
        </Setter.Value>
    </Setter>
</Style>

Screen Reader Support

ARIA Labels

Provide descriptive labels for all interactive elements.

Do

Use descriptive labels that convey purpose

<Button AutomationProperties.Name="Save patient record"
        Content="Save"/>

Don't

Rely on visual text or icons alone

<Button Content="✓"/>

XAML Accessibility Properties

XAML 
<!-- Automation Properties for Screen Readers -->
<Button Content="Add"
        Command="{Binding AddCommand}"
        AutomationProperties.Name="Add new patient"
        AutomationProperties.HelpText="Register a new patient in the system"/>

<!-- Live Regions for Dynamic Content -->
<TextBlock AutomationProperties.LiveSetting="Polite">
    <TextBlock.Style>
        <Style TargetType="TextBlock">
            <Setter Property="Text" Value="{Binding StatusMessage}"/>
        </Style>
    </TextBlock.Style>
</TextBlock>

<!-- Heading Levels for Navigation -->
<TextBlock Text="Patient Registration"
           AutomationProperties.HeadingLevel="Level1"/>
<TextBlock Text="Personal Information"
           AutomationProperties.HeadingLevel="Level2"/>

<!-- Descriptive Labels for Form Fields -->
<TextBox AutomationProperties.Name="Patient full name"
         AutomationProperties.HelpText="Enter the patient's full name as shown on ID"
         Text="{Binding PatientName}"/>

<!-- Landmark Roles -->
<Border AutomationProperties.AutomationId="MainContentRegion">
    <!-- Main content -->
</Border>

<Border AutomationProperties.AutomationId="NavigationRegion">
    <!-- Navigation -->
</Border>

Color and Contrast

Contrast Ratios

All text and interactive elements meet minimum contrast requirements.

Element Type Minimum Ratio Our Ratio
Normal text 4.5:1 14.5:1 ✓
Large text (18px+) 3:1 14.5:1 ✓
UI components 3:1 5.2:1 ✓
Graphical objects 3:1 4.8:1 ✓

Color Independence

Never rely on color alone to convey information.

Do

Use color + text + icons for status indicators

✓ Success - Operation completed

Don't

Use color as the only indicator

Success

Focus Indicators

Visible Focus

Keyboard focus must be clearly visible at all times.

Focus Order in Forms

Maintain logical focus order through form fields.

XAML 
<!-- Logical tab order through patient registration form -->
<Grid>
    <Grid.RowDefinitions>
        <RowDefinition Height="Auto"/>
        <RowDefinition Height="Auto"/>
        <RowDefinition Height="Auto"/>
        <RowDefinition Height="Auto"/>
    </Grid.RowDefinitions>

    <!-- Top to bottom, left to right -->
    <TextBox Grid.Row="0"
             TabIndex="0"
             AutomationProperties.Name="Patient ID"
             Text="{Binding PatientId}"/>

    <TextBox Grid.Row="1"
             TabIndex="1"
             AutomationProperties.Name="Patient Name"
             Text="{Binding PatientName}"/>

    <DatePicker Grid.Row="2"
                TabIndex="2"
                AutomationProperties.Name="Date of Birth"
                SelectedDate="{Binding DateOfBirth}"/>

    <StackPanel Grid.Row="3"
                Orientation="Horizontal"
                TabIndex="3"
                AutomationProperties.Name="Sex">
        <RadioButton Content="Male" GroupName="Sex"/>
        <RadioButton Content="Female" GroupName="Sex"/>
    </StackPanel>
</Grid>

Error Handling

Error Identification

Errors must be clearly identified and easy to understand.

⚠ Invalid format. Use P-XXXX-XXXX.
⚠ This field is required.

XAML Validation with Accessibility

XAML 
<!-- Accessible Form Validation -->
<TextBox x:Name="PatientIdField"
         Text="{Binding PatientId, UpdateSourceTrigger=PropertyChanged}"
         AutomationProperties.Name="Patient ID"
         AutomationProperties.HelpText="Enter patient ID in format P-XXXX-XXXX"/>

<!-- Error Template -->
<ControlTemplate x:Key="ValidationErrorTemplate">
    <StackPanel Orientation="Horizontal" Margin="4,0,0,0">
        <TextBlock Text="⚠"
                   Foreground="{StaticResource Semantic.Error}"
                   AutomationProperties.Name="Error indicator"/>
        <TextBlock Text="{Binding [0].ErrorContent}"
                   Foreground="{StaticResource Semantic.Error}"
                   FontSize="12"
                   Margin="4,0,0,0"/>
    </StackPanel>
</ControlTemplate>

<!-- Bind error template to text box -->
<Validation.ErrorTemplate>
    <StaticResource ResourceKey="ValidationErrorTemplate"/>
</Validation.ErrorTemplate>

<!-- Announce errors to screen readers -->
<Style TargetType="TextBox">
    <Style.Triggers>
        <Trigger Property="Validation.HasError" Value="True">
            <Setter Property="AutomationProperties.HelpText"
                    Value="{Binding (Validation.Errors)[0].ErrorContent}"/>
        </Trigger>
    </Style.Triggers>
</Style>

XAML Text Scaling Support

XAML 
<!-- Use relative sizing for text scalability -->
<UserControl>
    <UserControl.Resources>
        <sys:Double x:Key="FontSize.Base">14</sys:Double>
        <sys:Double x:Key="FontSize.Small">12</sys:Double>
        <sys:Double x:Key="FontSize.Large">18</sys:Double>
    </UserControl.Resources>

    <!-- Text will scale with system settings -->
    <TextBlock FontSize="{StaticResource FontSize.Base}"
               Text="This text scales with Windows settings"/>

    <!-- ViewBox for preserving layout at scale -->
    <ViewBox StretchDirection="Both" Stretch="Uniform">
        <Grid Width="800" Height="600">
            <!-- Fixed-size layout that scales -->
        </Grid>
    </ViewBox>
</UserControl>

Testing Checklist

Manual Testing

Keyboard Navigation Test

  • Can all interactive elements be reached via Tab?
  • Is the tab order logical and predictable?
  • Can all actions be performed via keyboard?
  • Is focus clearly visible on all elements?
  • Does Esc close modals and dialogs?

Screen Reader Test

  • Are all elements announced with clear labels?
  • Are form fields associated with their labels?
  • Are errors announced when they occur?
  • Is dynamic content announced via live regions?
  • Are heading levels used correctly?

Color and Contrast Test

  • Does text meet minimum contrast ratios?
  • Can information be understood without color?
  • Do focus indicators have sufficient contrast?
  • Is the interface usable in high contrast mode?
