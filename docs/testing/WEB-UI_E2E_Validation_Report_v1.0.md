# Web UI E2E Validation Report
## HnVue Console SW

---

## 목적

이 문서는 `feature/web-ui` 브랜치의 브라우저 기반 검증판에 대해 자동 클릭 방식 E2E 검증 범위를 정의하고, PRD/SAD/UEF 기준 대비 현재 웹 UI 구현이 어디까지 커버되는지 정리한다.

웹 UI는 실제 하드웨어 제어 제품이 아니라 데스크톱 GUI와 공용 모듈 설계의 완성도를 높이기 위한 검증 도구이며, 다음 목표를 가진다.

- 5클릭 표준 촬영 흐름의 반복 검증
- 위험 관련 사용 시나리오의 차단/복구 UX 검증
- 역할 기반 접근과 CD 배포 확인 절차 검증
- 한/영 전환과 응급 시작 UX의 회귀 방지

---

## 자동 E2E 범위

### E2E-001 표준 촬영 5클릭 완료

- 문서 기준: `PR-NF-UX-021`, `ST-WF-001`, `UEF US-001`
- 자동 시나리오:
  1. 로그인
  2. 흉부 환자 선택
  3. Chest PA Standard 선택
  4. 파라미터 확인
  5. 촬영 실행
  6. PACS 전송
- 기대 결과:
  - 클릭 계수 `5 / 5`
  - `Completed` 상태
  - PACS 큐 등록 로그 표시

### E2E-002 소아 프로토콜 미스매치 차단 및 복구

- 문서 기준: `HRUS-002`, `SAD-WF 상태머신`, `PR-NF-UX-025`
- 자동 시나리오:
  1. 소아 환자 선택
  2. 성인 Knee Trauma Standard 선택
  3. 차단 경고 확인
  4. ACK 후 Pediatric Knee Low Dose 재선택
  5. 파라미터 확인 가능 여부 확인
- 기대 결과:
  - 차단 상태 유지
  - 올바른 저선량 프리셋 복원 후만 다음 단계 허용

### E2E-003 응급 시작 3클릭 준비

- 문서 기준: `PR-NF-UX-026`, `UEF US-002`, `UEF UT-012`
- 자동 시나리오:
  1. 상단 `Emergency start` 클릭
  2. 파라미터 확인
  3. 촬영 실행
- 기대 결과:
  - 응급 환자와 권장 프로토콜 자동 적재
  - `3 / 5` 클릭 시점에 `ImageReview` 진입

### E2E-004 CD 배포 최종 확인

- 문서 기준: `MR-072`, `SAD-CD-1000`, `UEF UT-014`
- 자동 시나리오:
  1. Radiologist 로그인
  2. Delivery 진입
  3. 굽기 시작
  4. 환자/스터디 확인 다이얼로그 확인
  5. 확인 후 굽기
- 기대 결과:
  - 최종 확인 다이얼로그 표시
  - Completed 상태 및 감사 로그 표시

### E2E-005 역할 기반 접근 및 다국어 전환

- 문서 기준: `MR-033`, `PR-NF-SC-043`, `MR-045`
- 자동 시나리오:
  1. Admin 로그인
  2. `/console` 직접 진입 시도
  3. `/admin`으로 복귀 확인
  4. 언어 전환
- 기대 결과:
  - 권한 밖 화면 차단
  - Admin 화면 텍스트 실시간 갱신

## 구현 보강 사항

이번 웹 검증판 보강에서는 다음을 추가했다.

- 5클릭 목표를 수치로 확인할 수 있는 클릭 계수 카드
- 응급 환자/권장 프로토콜을 즉시 불러오는 `Emergency start`
- 치명 경고와 복구 경로를 안내하는 검증 가이드 카드
- CD 배포 전 환자/스터디를 다시 확인하는 PHI 확인 다이얼로그
- Playwright 기반 자동 클릭 회귀 테스트와 CI 연동

---

## 현재 해석

이 웹 검증판은 다음 범위에서 데스크톱 GUI 대비 높은 신뢰도의 검증 도구로 사용할 수 있다.

- 정보 구조와 역할별 화면 접근
- 표준 촬영 흐름의 클릭 수와 단계 전이
- 소아/경고 차단과 복구 UX
- 배포용 미디어 확인 흐름
- 언어 전환, 응급 시작, 감사 로그 피드백

다만 아래 항목은 여전히 mock 또는 시뮬레이션이다.

- 실제 Generator/Detector 제어
- 실제 DICOM 송수신 및 PACS 30초 성능
- 실제 인증/세션/하드웨어 드라이버
- 실제 Read-back verify 및 광학 드라이브 상태

## 실행 명령

```bash
cd /Volumes/work/console-gui/src/HnVue.Web
pnpm install
pnpm exec playwright install chromium
pnpm test:e2e
```

CI에서는 `Web UI CI`가 브라우저 설치와 E2E까지 포함해 동일한 회귀 검증을 수행한다.
