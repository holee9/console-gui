## 1. 화면 개요

촬영(Acquisition) 화면은 FPD (Flat Panel Detector) X-ray 촬영 워크플로우의 핵심 화면이다. 선택된 환자/오더에 대해 노출 파라미터를 설정하고, 실제 방사선 촬영을 실행하며, 취득된 영상을 실시간으로 확인하는 모든 기능이 단일 화면에 통합되어 있다.

**주요 기능:**
- 환자 정보 및 검사 정보 표시 (좌측 패널)
- 장비 상태 실시간 모니터링 (Generator / Detector / Bucky / Collimator)
- 노출 파라미터 설정 (kVp / mAs / SID / Grid / AEC)
- 촬영 프로토콜 선택
- DICOM 이미지 뷰어 (실시간 업데이트)
- 촬영 시작 / 취소 / 재촬영 / 삭제 제어
- 긴급 정지 (Emergency Stop) — IEC 62366 필수 기능
- 촬영 완료 썸네일 스트립

**설계 원칙:**
- 안전 중요 컨트롤(긴급 정지)은 항상 가시적이어야 하며 절대 다른 UI에 가려지지 않음
- 최소 터치 타겟 44px 준수, 핵심 버튼(촬영/정지)은 최소 44px
- 방사선 촬영 환경에 맞는 극단적 다크 테마 (이미지 뷰어 배경 `#050d18`)
- 상태 변화는 컬러 + 텍스트 이중 코딩 (색맹 사용자 고려)

### 3.2 긴급 정지 버튼 (Emergency Stop) — IEC 62366 핵심 요구사항

긴급 정지 버튼은 IEC 62366의 안전 중요 인터럭 기능이다.

#### 위치 및 가시성

| 항목 | 명세 |
|------|------|
| 위치 | 앱 헤더 우측 끝, 항상 고정 |
| 높이 | 앱 헤더 전체 높이 (30px) — 헤더를 가득 채움 |
| Margin | 헤더 상/하/우 패딩 보정 (음수 마진으로 헤더 끝까지) |
| Z-index | 항상 최상위 |
| 가시성 | **절대 숨김 불가, 다른 UI에 가려짐 불가** |

#### 버튼 스타일

| 속성 | 값 |
|------|----|
| 배경 | `#e74c3c` (safety.emergency) |
| 호버 배경 | `#ff2010` |
| 활성(Active) 배경 | `#cc1008` |
| 텍스트 | "■ STOP" (U+25A0 블랙 사각형 + 텍스트) |
| 폰트 | 12px / 700 / `#ffffff` |
| letter-spacing | 0.5px |
| Padding | 0 14px |
| 테두리 반경 | 없음 (헤더 끝에 맞춤) |
| 키보드 단축키 | **Escape** (항상 활성) |

#### 긴급 정지 동작 명세 (IEC 62366 §5.9)

```
긴급 정지 발동 시:
1. 현재 촬영 즉시 중단 (확인 다이얼로그 없음 — IEC 62366 요구)
2. Generator 노출 신호 즉각 차단
3. 촬영 시작 버튼 비활성화
4. 상태 오버레이: "긴급 정지 — 시스템 정지" (#e74c3c)
5. 상태 바: "긴급 정지" (#e74c3c)
6. 노출량 모니터 리셋 → 0.00 mGy
7. 버튼 텍스트: "정지됨", 배경 #7a0000 (진한 적색)
8. 시스템 리셋 후 자동 복구 (N초 후, 또는 수동 확인)
```

> **설계 금지사항:** 긴급 정지 버튼에 확인 다이얼로그를 추가하지 않는다. 확인 단계는 긴급 상황에서 지연을 유발하여 더 큰 위험을 초래한다 (IEC 62366-1:2015 §5.9.2).

## 6. 접근성 및 안전성 고려사항 (IEC 62366)

### 6.1 IEC 62366 의료기기 사용성 공학 요구사항

#### 6.1.1 안전 관련 사용 오류 방지

| 위험 시나리오 | 예방 설계 |
|------------|---------|
| 잘못된 환자에게 방사선 조사 | 촬영 전 환자 정보 좌측 패널 상단 고정 표시, Patient ID 강조색 |
| 의도치 않은 촬영 시작 | 촬영 버튼과 취소 버튼 시각적 분리, 취소 버튼은 호버 시에만 적색 |
| 긴급 상황에서 정지 버튼 탐색 실패 | 헤더 우측 끝 고정 위치, 항상 가시, 적색 + 큰 터치 타겟 |
| 고선량 경고 무시 | 경고 다이얼로그 황색 헤더 (`#f39c12`), 적색 아닌 황색으로 경고 구분 |
| 장비 오류 시 촬영 시도 | 장비 상태 비정상 시 촬영 버튼 자동 비활성화 |

#### 6.1.2 터치 타겟 요구사항 (IEC 62366 + 글러브 사용 고려)

   - 환자 ID 확인
   - 장비 상태 체크
   - 위험 선량 경고

### Phase 2 — 핵심 워크플로우
