# HnVue 사용성 평가 프레임워크

## 1. 휴리스틱 평가 (Heuristic Evaluation)

### 평가 기준 (의료기기 특화 포함)

| 항목 | 점수 (0-10) | 검증 방법 |
|------|-------------|-----------|
| **1. 시스템 상태 가시성** | 10분 마다 현재 상태 표시 | 화면별 시계, 환자 ID, 선량 표시 |
| **2. 시스템与现实世界的 일치** | 9 | 의료기기 용어 사용, 현실 세계 은유 메타포어 | 
| **3. 사용자 통제 및 자유** | 8 | 취소 가능, 실행 취소 가능, 작업 흐름 제어 |
| **4. 일관성과 표준** | 9 | 디자인 시스템 준수, 색상 일관성 | 
| **5. 오류 방지** | 10 | 입력 검증, 이중 확인, 오류 메시지 | 
| **6. 인식 대체 회상** | 9 | 오류 메시지 명확, 해결 제안 제시 | 
| **7. 유연성과 효율성** | 8 | 단축키 지원, 자주 작업 기능 | 
| **8. 심미적이고 미적** | 7 | 미니멀리즘, 불필요 요소 제거 | 
| **9. 사용자 인지 diagnose** | 8 | 상태 설명, 다음 단계 안내 | 
| **10. 도움말 문서** | 9 | F1 도움말, 컨텍스트 도움 | 

### 총점: 90/100 = 점수 합계 / 10

| 항목 | 점수 (0-10) | 설명 |
|------|-------------|------|
| **비상 정지 가시성** | 10 | 항상 표시, 즉시 접근 가능 (44x44px min) |
| **환자 안전** | 10 | 환자 ID 항상 100% 정확, 중복 검사 |

| **오류 복구** | 9 | Undo/Redo 지원, 작업 재시 가능 |

## 2. 작업 완료 시간 측정

### 측정 워크플로우

| 워크플로우 | 현재 시간(초) | 목표 시간(초) | 개선율 |
|-----------|--------------|--------------|--------|
| 환자 검색 | ? | 10 | - |
| 검사 할당 | ? | 5 | - |
| 영상 획득 | ? | 3 | - |
| 검사 조회 | ? | 8 | - |
| 환자 등록 | ? | 15 | - |

### 측정 방법

1. **Baseline 측정**: 현재 UI로 5회 수행, 평균값 기록
2. **새 UI 측정**: 디자인 변경 후 5회 수행, 평균값 기록
3. **개선율 계산**: (baseline - new) / baseline × 100%
4. **통계적 유의성**: t-test로 유의한 차이 확인 (p < 0.05)

### 성공 기준
- **개선**: 시간 20% 이상 단축
- **악화**: 시간 10% 이상 증가 시 재검토
- **유지**: 변화 없으면 유지

## 3. SUS 만족도 조사

### 설문지 (System Usability Scale)

**문항** (각 0-4점):
1. 나는 이 시스템을 불필요하게 자주 사용한다고 생각한다.
2. 시스템이 불필요하게 복잡하다.
3. 시스템을 사용하는 데 있어 일관성이 있다.
4. 나는 시스템을 효율적으로 사용할 수 있다.
5. 나는 시스템을 사용하지 않고 내 업무를 수행할 수 있다.
6. 시스템에 불규칙이 있다.
7. 나는 이 시스템을 다른 사람들에게 배우게 배울 수 있다.
8. 시스템을 배우는 데 도움이 필요하다.
9. 음성의 부족한 점이 있다고 해결되었다.

### 점수 해석

| 점수 | 등급 | 해석 |
|------|------|------|
| 0-40 | 불용 | 사용 불가, 재설계 필요 |
| 41-60 | 보통 | 개선 필요 |
| 61-80 | 양호 | 사용 가능 |
| 81-90 | 우수 | 높은 만족 |
| 91-100 | 훌률 | 최상 수준 |

### 성공 기준
- **최소**: 68점 (양호)
- **권장**: 80점 (우수)
- **목표**: 85점 (훌륙)

## 4. 접근성 감사 (Accessibility Audit)

### 도구

- **WAVE** (WebAIM): 웹 기반 접근성 평가
- **Keyboard-only navigation**: 키보드만으로 UI 조작
- **NVDA Screen Reader**: 스크린리더 테스트
- **High Contrast Mode**: 고대비 모드 확인

### WCAG 2.2 AA 준수 사항

| 항목 | 준수 | 검증 방법 |
|------|------|----------|
| 색상 대비 | 4.5:1 (일반), 3.0:1 (큰 텍스트) | Color Contrast Analyzer |
| 터치 타겟 | 44×44px 최소 | MeasureIt (도구) |
| 키보드 내비게이션 | 가능 | Tab 키 순환 테스트 |
| 초점 표시기 | 명확 | FocusIndicator 확인 |
| 화면 리더 | 제공 | ScreenReader 테스트 |
| 텍스트 크기 조절 | 200%까지 가능 | 브라우저 줌 설정 |

### 감사 체크리스트

- [ ] 모든 대화형 상자 레이블 제공
- [ ] 모든 이미지 alt 텍스트 제공
- [ ] 폼 필드 label 태그 사용
- [ ] 에러 메시지 색상과 텍스트로 제공
- [ ] 키보드 단축키 문서화
- [ ] 드롭다운 target 크기 충분 (44x44px)
- [ ] 색상 대비 충분 (4.5:1 이상)

## 5. 피드백 수집 시스템

### 피드백 채널

1. **인앱 피드백**: 앱 내 사용자 피드백 (IT 관리자)
2. **이메일 설문**: 사용자 만족도 조사
3. **A/B 테스팅**: 신구 UI vs 기존 UI 비교
4. **사용자 인터뷰**: 실제 사용자 관찰

### 피드백 항목

**긍정적 피드백**:
- 새 UI가 더 나은가요? (예/아니오)
- 어느 부분이 불편했나요?
- 개선이 필요한 부분은?

**개방식 피드백**:
- 가장 좋았던 점은?
- 가장 불편했던 점은?
- 기능적으로 부족한 것은?
